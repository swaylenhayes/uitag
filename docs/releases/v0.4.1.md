# v0.4.1 Release Notes

Bugfix and UX polish release following v0.4.0.

## Bug Fixes

### Rescan Special-Character Guard

The rescan replacement gate now requires the new reading to preserve at least as many special characters as the original. Previously, the gate only checked confidence — a high-confidence but sanitized crop result could replace a lower-confidence but more correct reading.

**Examples of regressions this prevents:**
- `;([\w_]+);` → `;([w_]+);` — backslash dropped, confidence went 0.30→1.0
- `;([\w_]+);` → `;([\W_]+);` — lowercase `w` became uppercase `W`
- `TAG=[%Variable%Local_Tag%]` → `ТAG=[%Variable%LocaI_Tag%]` — Cyrillic `Т` substitution, `l`→`I` swap

### Yellow Bounding Box Contrast

Replaced yellow `(255, 255, 0)` with dark gold `(200, 170, 0)` in the SoM annotation palette. Yellow was nearly invisible on light mode backgrounds (luminance ~226, close to white). Dark gold is clearly visible on both light and dark backgrounds.

## CLI UX Improvements

### Interactive Rescan Prompt

When low-confidence detections are found in a TTY, the CLI now prompts:

```
4 low-confidence detection(s):
  [10] CONF 0.30  ";([\w_]+);"
  [32] CONF 0.50  "v →"
  [38] CONF 0.50  "V"
  [43] CONF 0.50  "TAG=[%Variable%Local_Tag%]"

Note: dark background detected — light mode screenshots
typically produce more accurate OCR for code and special characters.

Rescan low-confidence detections? [y/N]
```

Typing `y` re-runs the pipeline with rescan and saves output with `-rescan` suffix. In non-TTY environments, falls back to `Tip: run with --rescan`.

### Output Changes

- **Bold orange header** for low-confidence callout (ANSI `\033[1;33m`)
- **Reformatted list**: `[id] CONF 0.xx  "output"` (was `[id]  "output"  conf: 0.xx`)
- **`-rescan` filename suffix**: `--rescan` outputs are now `{stem}-uitag-rescan.png` and `{stem}-uitag-rescan-manifest.json`, preserving the standard output files
- **Dark mode hint** repositioned to appear after the low-confidence list, before the rescan prompt
